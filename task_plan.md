# kruon MVP PRD 调研与交付计划

## 目标

结合参考视频、项目路线图、品牌资料与脑暴文档，先完成外部调研和假设甄别，再输出一份可供产品、设计与开发直接执行的 MVP 产品需求说明书（DOCX）。

## 阶段

| 阶段 | 状态 | 产出 |
|---|---|---|
| 1. 资料盘点与证据提取 | complete | 视频、路线图、品牌与脑暴材料摘要 |
| 2. 外部调研 | complete | 协议、竞品、技术边界与市场证据 |
| 3. 假设甄别与 MVP 收敛 | complete | 采纳/验证/暂缓矩阵，MVP 边界 |
| 4. PRD 内容设计 | complete | 用户、流程、功能、数据、安全、指标、验收与计划 |
| 5. DOCX 生成 | complete | kruon_MVP产品需求说明书_2026-07-11.docx |
| 6. 渲染与质量验收 | complete | 26 页逐页检查、表格/标题/可访问性/残留检查通过 |

## 关键决策

- 脑暴文档只作为假设库，不直接视为事实或需求。
- PRD 采用 `compact_reference_guide` 设计预设和 `memo_masthead` 首屏结构。
- 优先定义一个可验证的端到端闭环，而不是把路线图中所有能力压入 MVP。
- 外部调研优先使用官方文档、协议规范和产品一手资料。
- PRD 的 P0 以“可被人工验收的任务闭环”为边界；P1 只列出可独立验证的增强项，不混入 Alpha 必交付。
