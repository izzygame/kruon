# W1 运行核心安全审查

- 日期：2026-07-14
- 审查者：Hermes（`deepseek-v4-flash-260425`）与 Codex
- 范围：进程治理、SQLite 一致性、崩溃恢复、路径/环境/持久化/IPC 失败模式

## 结论

W1 技术闭环达到“受控本地只读探针”门槛，但不具备操作系统级完全隔离。有效问题已修复；无法在 W1 可靠解决的边界已进入 ADR，不以文档措辞掩盖。

## 审查处置

| 审查项 | 处置 | 结果 |
|---|---|---|
| 子进程继承全部宿主环境 | 采纳 | `env_clear` + 固定白名单；测试证明未列入的 secret 不可见 |
| Tauri IPC 返回内部错误全文 | 采纳 | 映射为稳定公开错误，不泄漏路径/数据库详情 |
| 脱敏键覆盖不足 | 采纳 | 扩展 token/secret/password/key/credential/authorization 后缀与类别 |
| stdin 可能未关闭 | 误报但加固 | Rust 作用域原本会关闭；增加显式 `drop` 与必须等待 EOF 的测试 |
| cancel 与 wait 双重收口 | 已有保护并补证 | 增加并发竞态测试，得到同一稳定终态 |
| 单 SQLite 连接是 P0 | 降级 | W1 选择保守串行一致性；连接池作为可测吞吐问题再处理 |
| `uncertain` 输出线程无法 join | 接受剩余风险 | 不能在未确认进程退出时阻塞 join；管理对象保留，后续加入 reaper |
| `setsid`、网络出口、运行期路径替换 | 接受剩余边界 | 明确不宣称已隔离；进入 ADR-004 的后续安全门 |
| 真实探针尚未执行 | 信息过期 | Hermes 基于合并提交审查；探针随后已各执行一次并保存脱敏证据 |

## 失败关闭原则

- 路径在启动前拒绝。
- 事件冲突、跳号或终态后追加报错，不猜测覆盖。
- 重启后非终态 Run 不操作旧 PID，转为 `uncertain`。
- 无法确认完整进程组清理时不宣称 `cancelled`，转为 `uncertain`。
