# M2 任务控制闭环验收记录

日期：2026-07-16

## 交付范围

M2 将 M1 的只读执行面扩展为可审计的任务控制闭环：

- `RunSnapshot` 保存冻结启动计划指纹；它覆盖适配器、命令、参数、工作目录、提示词哈希与环境变量名，不保存提示词正文。
- 新增独立的 `M2Store`，持久化审批请求/决定、Artifact、完成报告、任务验收和控制审计记录。
- 审批模型绑定 `run_id`、动作类型与原始参数指纹；审批决定要求精确指纹，缩小范围会使原请求进入 `superseded` 并产生需要重新批准的新指纹；过期请求会成为 `expired` 并写入审计。
- 当前 Codex 与 Claude 固定启动计划都如实标记为 `sandbox_policy_only`、`read_only`。没有已验证的逐动作请求/响应传输，因此 UI 不提供虚假的批准按钮；`M2Store` 的 `per_action` 路径仅为未来实际具备该能力的适配器保留，且拒绝把 `sandbox_policy_only` 冒充为逐动作审批。
- Run 详情提供取消、暂停能力说明、恢复建议和“新建跟进 Run”。暂停/会话续接均不作未验证承诺；跟进始终重走新的只读启动计划。
- Artifact 收集包括适配器报告的工作区文件、diff 指纹和受管 Run 的终态诊断；不复制原始 CLI 日志或 diff 正文。文件路径必须经工作区及 Task scope 校验，且不会为计算内容哈希再次读取文件，从而避免审阅时的符号链接置换内容读取。
- 完成报告中的路径必须是现有父目录下、处于允许 scope 内的相对路径；Task 只有在 Run `completed` 且已记录完成报告后才能人工接受。退回必须填写说明。接受、退回、Artifact、报告和跟进排队均会产生可查看的 Run 审计记录。

## 控制台

桌面控制台的 Run 详情区展示：

- 冻结启动指纹和真实的审批/暂停能力边界；
- 取消、Artifact 收集、恢复建议与新跟进 Run；
- 结构化完成报告（摘要、检查结果、相对改动路径）；
- 人工接受/退回与 Run 审计轨迹。

本地 Vite 预览已检查渲染和浏览器诊断。预览未启动 Tauri 后端，因而显示的“runtime unavailable”是预期的本地 Web 预览状态，不是产品运行时失败。

## 发布门说明

M2 的实现与本地自动化闭环已交付，但 MVP 计划中的统计性发布门尚未完成：目前没有 20 组真实 Codex/Claude 任务的启动率和终态率样本，也没有可验证的生产级 Claude 逐动作权限回传。因此不能把该统计门或逐动作执行安全性标记为已验收。

在当前实际支持的只读、冻结策略模式中，未证明安全的动作不会被执行或被静默放行；后续若接入真实逐动作适配器，必须先完成双向传输验证，并以 M2 审批绑定、拒绝与重新审批测试重新验收。
