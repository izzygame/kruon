# M3.1 Kenney visual asset verification

- Date: 2026-07-16
- Scope: selected local visual assets for the optional M3 world window
- Decision: ADR-007 amendment
- Outcome: implemented and verified in the Windows release build

## Selection and license

The repository does not vendor either complete pack. It keeps only six Furniture Kit v1.0 GLBs, two Mini Characters v1.0 GLBs, the shared character colormap, and each pack's original license file.

- Furniture source: https://kenney.nl/assets/furniture-kit
- Character source: https://kenney.nl/assets/mini-characters
- License source: https://kenney.nl/support
- License: Creative Commons CC0; attribution is not required
- Exact files, sizes, triangles, SHA-256 values, and conversion record: `apps/desktop/public/assets/world/kenney/ASSET_MANIFEST.md`

The nine runtime files total 606,458 bytes. The two-station layout renders approximately 3,581 imported triangles and one shared 512 x 512 indexed PNG. The original GLBs and PNG are stored byte-for-byte; the full ZIP archives and every unselected asset remain outside the repository.

## Runtime design

- `KenneyOffice.tsx` is a second lazy chunk under the already-lazy `WorldScene` renderer.
- All URLs start with the packaged `/assets/world/kenney/` path. CSP permits `'self'` fetches but no remote model origin.
- Furniture clones share loaded resources. Skinned characters use skeleton-safe clones.
- Each character samples the bundled `idle` clip once to avoid a bind pose; there is no animation loop, and the Canvas retains `frameloop="demand"`.
- State color rings and markers still derive from the Rust `WorldSnapshot`; models do not own or infer runtime state.
- Suspense and an asset-specific error boundary render the original procedural stations while assets load or whenever the selected layer fails.

## Release evidence

The Windows release build rendered both Kenney characters, desks, office chairs, displays, keyboards, mice, the shared colormap, status rings, and the potted plant. Both stations remained `sleeping` because the local database contained no Runs. The main 2D control window remained authoritative and usable.

| Measurement on DRAGON, 16 logical processors | Procedural M3 | Kenney M3.1 |
|---|---:|---:|
| Process tree | 8 | 8 |
| Working set | 489.0-497.1 MiB | 511.8 MiB |
| Private memory | 298.7-306.9 MiB | 311.3 MiB |
| CPU, 5-second sample normalized to total logical-CPU capacity | 0.57-0.98% | 0.90% |

The observed Kenney increment was 14.7-22.8 MiB of working set and 4.4-12.6 MiB of private memory. The CPU sample stayed inside the previous short-sample range. These values are local regression evidence, not cross-device guarantees.

## Automated evidence

- `assets.test.ts` freezes local-only URLs, hashes, a 606,458-byte payload, and 3,581 visible imported triangles.
- Desktop frontend suite: 3 files, 10 tests passed.
- TypeScript check and production Vite build passed.
- Production chunking keeps `KenneyOffice` separate from the main UI and from `WorldApp`; binary assets are fetched only by the world renderer.

DEV-307 recruited-participant evidence remains open. Visual polish does not substitute for proof that the world improves task understanding.
