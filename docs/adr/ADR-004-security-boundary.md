# ADR-004：工作区、环境、存储与 IPC 安全边界

- 状态：已采纳
- 日期：2026-07-14

## 背景

本地模型 CLI 可读取宿主机环境与文件。W1 必须先限制明显越界和持久化泄漏，同时准确声明尚未实现的系统级隔离。

## 决策

- 启动前 canonicalize 工作区和工作目录，拒绝 `..`、绝对路径越界和符号链接逃逸。
- 子进程执行 `env_clear`，仅传递运行和本地登录所需的固定环境白名单；API key、SSH agent 和云凭据变量不继承。
- SQLite 仅存归一化、递归脱敏事件；畸形原始行只存 SHA-256；提示只存哈希。
- Tauri IPC 返回稳定错误类别，不返回数据库、路径或子进程内部错误文本。
- 真实探针只允许系统临时目录中的普通、非符号链接夹具，并比较运行前后文件指纹。

## 明确未覆盖

- 尚未实现网络出口隔离。
- canonicalize 之后到 CLI 读取期间仍存在路径替换 TOCTOU 窗口。
- 暂未从模型输出中识别并再次验证“产物路径”。
- Unix 进程可通过 `setsid` 逃离原进程组。
- PID/PGID 残留检测可能受 PID 复用影响，因此不确定时失败关闭为 `uncertain`。

## 未选择方案

W1 不引入容器、虚拟机、macOS Seatbelt profile 或全局网络代理。它们需要单独的兼容性与产品授权设计。

## 复审条件

进入可写工具、自动审批或外部发布能力前，必须先复审并优先加入系统级沙箱与出口策略。
