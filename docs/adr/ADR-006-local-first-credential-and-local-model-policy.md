# ADR-006：local-first、凭据保管与本地模型离线语义

- 状态：已采纳
- 日期：2026-07-15

## 背景

Kruon 的产品定位需要同时表达用户自主、本地数据掌控、云端工具可选接入和未来的完全本地执行能力。若把这些主张压缩成“无需登录、不联网、API key 本地配置”，会掩盖三个事实：云端工具仍各自需要认证与网络；W1 尚未有本地模型适配器；将密钥写入项目配置或继承给所有子进程会破坏 ADR-004 的安全边界。

本决策定义可以对外承诺的边界，并为后续本地模型与 Provider 凭据功能设定不可绕过的实现约束。

## 决策

### 1. 用户账户与数据位置

- Kruon 不要求用户登录 Kruon 运营的账号系统即可创建本地 Workspace、使用本地控制平面或查看本地历史。
- Workspace、SQLite 事件、产品设置和凭据引用默认保留在设备本地；不得引入隐式云同步、遥测回传或集中式凭据服务。
- 用户接入的 Codex、Claude Code 或其他云端 Provider 仍可能要求其各自的登录、订阅、API key 和网络连接。Kruon 不替代也不伪造这些授权。

### 2. 执行模式的诚实表达

Kruon 将把执行器区分为以下模式：

| 模式 | 说明 | 是否可标记为“离线” |
|---|---|---|
| 云端 CLI / Provider | 本机启动 CLI，但模型或服务在外部 Provider 侧执行 | 否 |
| 本地模型 Provider | 模型进程和推理端点均由用户在本机管理 | 仅在满足下述离线验收后 |
| 未知 / 降级 | 网络行为、Provider 端点或能力尚未验证 | 否 |

“本地优先”不表示所有 Run 均离线。“完全本地化”只可用于模型进程、推理端点和所需工作流均可在本机完成，且没有未声明外部依赖的执行链路。

### 3. 凭据保管与注入

- API key、OAuth token、SSH 凭据和 Provider secret 不得写入项目文件、SQLite、事件、日志、诊断包或可读配置镜像。
- 凭据只能由操作系统安全存储保管（首发 macOS Keychain；Windows 版本使用等价的系统凭据存储），UI 只保存 Provider 标识和凭据引用。
- 子进程继续默认 `env_clear`。只有用户已为特定 Provider 显式配置凭据、该 Provider 的受控适配器证明确有需要时，才可在 Run 生命周期内进行最小范围的临时注入。
- 不能安全注入的凭据不得回退为全局环境继承；产品应要求用户在上游 CLI 的受支持认证方式中完成登录，或明确报告该 Provider 不受支持。
- 所有事件、错误、输出和诊断仍须使用既有脱敏规则，且要为新 Provider 的 secret 格式补充测试。

### 4. 本地模型路线

- 本地模型不是 W1/MVP 已交付能力；在适配器、连接设置、健康检查和 UI 完成前，不得在产品中宣称已支持特定运行时（例如 Ollama、llama.cpp 或 LM Studio）。
- 每个本地模型 Provider 必须拥有 capability manifest，声明端点类型、认证方式、允许的网络范围、模型发现方式和离线证据。
- UI 只能在以下条件全部满足时显示“离线”：模型进程/端点是本机资源；Provider 未声明外部依赖；适配器健康检查通过；运行时网络策略或等效的可验证控制阻止非本地 Provider 访问。
- 在现有“无网络出口隔离”的 W1 状态下，本地模型 Run 只能显示“本地端点，离线未验证”，不能显示“完全离线”。

### 5. 开源发布

代码公开不等于已有开源许可。对外使用“开源软件”表述前，项目必须选择许可证、提交 `LICENSE`，并在 README 中链接该许可证。许可证选择是产品/法律决策，不由本 ADR 指定。

## 后果

- 产品定位可以明确“无 Kruon 账号、数据本地优先、用户自选工具”，同时不误导用户以为云端 Provider 无需认证或网络。
- 凭据设置成为独立的安全功能，而不是把 key 写进项目配置的便利功能；其实现必须复审 ADR-004 的环境白名单和脱敏测试。
- 本地模型支持需要专门的适配器、离线判定和网络控制工作，不能作为现有 Codex/Claude 集成的营销别名。
- Open-source 定位在许可证落地前只能表述为发布方向或代码公开状态。

## 验收与复审条件

在交付任一本地模型或 Provider 凭据功能前，必须满足：

1. Provider capability manifest、凭据引用模型和设置 UI 的实现及测试齐备；
2. key 不写入项目、SQLite、事件、日志、诊断包，且子进程环境测试覆盖允许与拒绝路径；
3. 本地端点、外部端点和未知端点的 UI 标签与实际网络行为一致；
4. 若要使用“离线”标签，具备可重复的端到端离线验证和非本地网络阻断证据；
5. 对外宣称开源前，许可证文件和 README 许可链接已经落地。

本 ADR 补充 [ADR-004](ADR-004-security-boundary.md)，不放宽其 `env_clear`、脱敏或网络出口风险声明。
