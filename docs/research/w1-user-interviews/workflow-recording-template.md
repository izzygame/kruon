# 工作流录屏模板（15～20 分钟）

## 准备

使用无敏感信息的示例仓库或参与者自行准备的公开/虚构任务。关闭通知，确认屏幕中没有密钥、客户数据、私人聊天和个人路径。主持人只提示任务目标，不指导工具选择。

## 任务

“请像平时一样，让一个编码代理只读分析这个项目中的指定问题；查看过程，必要时中止任务；随后说明你会如何确认结果、恢复记录，并决定下一步。”

## 观察记录

| 时间 | 用户目标 | 所在工具/窗口 | 采取动作 | 犹豫或失败 | 需要的证据 | 原话摘要 |
|---|---|---|---|---|---|---|
| | | | | | | |

重点标记：代理选择、上下文复制、路径确认、审批、等待、取消、残留确认、任务恢复、结果比较和手工记账。

## 结束追问

1. 哪一步最不确定？
2. 哪一步最浪费时间？
3. 你认为任务何时真正结束？依据是什么？
4. 如果 Kruon 自动记录一个证据，你最希望是什么？
