# kruon MVP 开发规划

## 目标

把已审查通过的 MVP PRD 转换为可以直接排期、分工和验收的开发计划，覆盖技术预研、架构骨架、纵向闭环、3D 世界视图、Alpha 工程化及发布门槛。

## 当前约束

- 当前工作区没有产品代码，开发从零启动。
- 首发 macOS，P0 深适配 Codex CLI 与 Claude Code CLI。
- 2D 控制面是完整主路径；3D 是同源事件驱动、可关闭的状态视图。
- 本地优先、最小权限、参数绑定审批、人工验收是发布门槛。
- 计划需允许 Codex 主导复杂主线，并把边界清晰的支线拆给 Claude Code CLI。

## 阶段

| 阶段 | 状态 | 产出 |
|---|---|---|
| 1. 工作区与 PRD 复核 | complete | 输入、约束、空仓状态与计划口径 |
| 2. 技术预研与交叉审查 | complete | 适配器、安全、事件模型与技术选型结论 |
| 3. 工作分解与依赖建模 | complete | Epic、任务、依赖、并行泳道、负责人建议 |
| 4. 里程碑与验收门 | complete | 10 周排期、每阶段退出标准、测试与风险门 |
| 5. 开发计划交付 | complete | 可执行开发计划文档与首个迭代任务包 |

## 决策原则

- 先用 spike 消除不可逆技术风险，再固化架构。
- 每个阶段必须形成可演示、可测试、可回退的纵向增量。
- Claude Code 的输出是支线建议或验证证据，主方案由 Codex 复核整合。
- 不把 P1、远程 Agent、自动路由、云同步或复杂 3D 经营带入 P0。

## 错误记录

| 错误 | 尝试 | 处理 |
|---|---|---|
| Claude Code 只读审查长时间未返回输出 | 1 | 不阻塞主线，继续规划并保留会话轮询；若无结果则终止后缩小问题重试 |
