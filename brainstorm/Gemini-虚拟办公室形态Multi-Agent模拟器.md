# 📄 项目前期立项与可行性分析文档 (PRD 前置篇)

## 1. 项目定位与核心价值

* **产品名称（暂定）**：`AI Commander` / `Nexus Core` / `CyberOffice AI`
* **核心定位**：一款面向独立开发者、重度 AI 用户的**本地客户端 AI 资产调度中心与 3D 游戏化仪表盘**。

* **主要卖点 (USP)**：

* **3D 游戏化降维打击**：将枯燥的配置、命令行、多窗口脑暴，具象化为 3D 办公室。AI 在里面“走动”、“开会”、“敲代码”，满足人类的掌控欲与情绪价值，天然自带病毒式传播属性。

## 2. 行业背景与痛点分析 (Why Now?)

1. **客户端化与碎片化趋势**：如你所言，Codex、OpenClaw、Workbuddy 等本地客户端及各类 CLI/API 中转工具正成为主流。用户手里同时握着多个订阅（GPT、Gemini、MJ）和各种免费/付费 API（火山、DeepSeek）。
2. **“痛点”——心智成本高**：切换工具、手动同步上下文（把 GPT 的创意复制给 Claude 写代码）极其低效。
3. **“痒点”——缺乏视觉反馈与成就感**：传统的 AI 交互全是枯燥的 Chat 界面或 Terminal 窗口。用户花费巨资配置了顶级 AI 矩阵，却像在面对冷冰冰的 Excel 表格。

## 3. 产品核心功能模块 (3D游戏化 + 实用控制台)

整个产品采取 **“前店后厂”** 的架构：**前端是 3D 模拟游戏，后端是高效率的 AI 调度引擎。**

### 3.1 后端：本地控制中心 (The Engine)

* **万能 API & 订阅中转站**：支持导入自定义 API（OpenAI, Anthropic, DeepSeek, 火山引擎, 混元等）以及第三方工具的 Hook（对接你的现有工具）。
* **统一记忆体 (Shared Context Pool)**：在本地建立一个轻量级向量数据库（如 Chroma/SQLite）。当 Gemini 在“创意区”聊出结果后，该上下文自动同步进记忆池，Codex 角色在“开发区”写代码时能直接引用。
* **群聊脑暴编排器**：用户发布一个任务，系统根据预设的工作流，自动调度不同的模型（如：Gemini 提想法 -> GPT 评估 -> Claude 润色），并在前端 3D 界面以“开会”的形式展现。

### 3.2 前端：3D 虚拟工作室 (The View)

* **低成本 3D 渲染**：基于 WebGL/Three.js 或 Godot（打包为轻量本地客户端，如 Electron/Tauri 壳）。采用低多边形（Low-Poly）或体素（Voxel）网格风格，研发成本低且极具现代感。
* **AI 具象化 (NPC 映射)**：
* 每一个配置的 AI 账号/API 都是一个 3D 小人（NPC），头顶带有特定的 Model 标签（如截图中的 `Atlas | gpt-5.5`）。
* 支持用户自定义小人外观（甚至支持上传 `.glb` 格式的 3D 模型）。


* **游戏化办公行为映射**：
* **脑暴时**：几个大模型小人走到“会议室”坐下，头顶冒出对话气泡，实时滚动它们的回答和互相驳斥。
* **开发时**：负责代码的 NPC（如你的火山 Claude 替补）走到电脑桌前坐下，播放“敲键盘”动画，代表它正在后台执行 `claudecode` 任务。
* **额度耗尽/待机时**：小人会“躺平”、“喝咖啡”或显示为灰色“下班状态”，直观提示用户额度不足。

## 4. 工程体量与技术选型评估 (可控性)

正如你所规划的，**“不做底层 Agent”** 让这个项目的工程量从“地狱级”降到了“独立开发者可控级”：

* **技术栈建议**：
* **外壳/本地逻辑**：`Tauri` (Rust + Frontend) 或 `Electron`。Tauri 内存占用极小，适合做常驻后台的控制中心。
* **3D 前端引擎**：`Three.js / React Three Fiber`（纯 Web 方案，易于嵌入客户端）或你熟悉的 `Godot`（导出为库/独立组件）。鉴于你的背景，使用 **Godot 开发 3D 部分并通过 WebSocket 与本地后端通信** 是个绝佳的选择，能做出极其精致的 UI 和特效。
* **工作流引擎**：采用轻量级节点编排逻辑，或者直接本地集成开源的轻量级工作流脚本。


* **核心开发难点**：多模型并发时的 Token 流式传输（Streaming）与 3D 气泡框打字机效果的平滑同步。
